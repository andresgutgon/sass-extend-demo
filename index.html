Sass @extend from a css class with unexpected result

Blog post:

I want:

I want to NEVER style in blue the

inherits styles from a .

The problem:

In DEMO 1 sass code adds .base-class--in-red in generated CSS code under .is-blue parent div.

This was unexpected to me . Maybe it's only me but I didn't know that extending from another class your new class is extended across your entire source code.

In DEMO 2, extending from a %placeholder there is not that problem

SASS code: extending from .base-button


.base-button
  background-color: #f0f0f0
  border-radius: 20px
  border: 1px solid #ccc
  color: #333
  padding: 10px
  font-size: 13px
  margin: 0 10px 0 0

.base-button--in-red
  @extend .base-button
  background-color: #AD210E
  color: white
  border-color: #AD210E

// I only want my base button in blue
.blue-parent
  .base-button
    color: white
    background-color: darken($selected_color, 10%)
    border-color: darken($selected_color, 10%)

Generated CSS code:


// Here is the problem.
// base-button--in-red also pick BLUE styles
.blue-parent .base-button,
.blue-parent .base-button--in-red {
  color: white;
  background-color: #197e8d;
  border-color: #197e8d;
}
